## 二、统一业务场景体系（7+1 类）

> 说明：基于基础版文档中的 20 个细分场景进行归类，按**系统底层操作流程**来组织，便于逐个场景做端到端联调和功能测试。

### 场景1：产品与客户支持知识库助手

**覆盖原始行业**：SaaS 帮助中心、电商产品客服、客户成功团队、房地产/物业、政务智能问答  
**典型客户**：SaaS 公司、电商品牌、跨境卖家、物业公司、房产中介、政府/公共服务机构

**知识库内容示例**：
- 产品使用手册、API 文档、FAQ、版本更新日志、故障排查指南
- 退换货政策、物流说明、服务协议
- 房源信息、户型图、小区介绍、物业手册
- 办事指南、政策法规、表格模板、常见问题

**核心系统流程（Flow A）**：
- Admin UI (5173 或 9621/webui)：上传上述文档 → RAG Core(9621) 解析与向量化（Ollama Embedding）  
- Admin UI：在 Graph / Retrieval 视图中检查文档是否被正确索引、检索是否命中  
- Chat UI (3000)：终端用户提问 → Agent Service(2025) 通过 MCP(8001) 调用 RAG Core 检索 → Gemini 生成基于文档的回答

**代表需求**：
- "AI customer support bot for SaaS"
- "AI FAQ / knowledge base for e‑commerce"
- "Government / public service Q&A chatbot"
- "Real estate / property information assistant"

---

### 场景2：法律、合同与合规审查助手

**覆盖原始行业**：法律合同审查、供应链与采购、部分保险理赔/核保场景  
**典型客户**：律所、企业法务部、合规团队、采购/供应链部门、保险公司

**知识库内容示例**：
- 合同模板与真实合同（劳动合同、采购合同、保密协议等）
- 法律法规 PDF（合同法、劳动法、行业监管规定）
- 风险条款清单、内部合规制度、供应链采购制度
- 保险条款、核保规则、理赔案例

**核心系统流程（Flow A + Flow C）**：
- Admin UI：上传合同、法规和风险清单 → RAG Core 解析与索引  
- Admin UI：通过 Retrieval/Graph 检查特定条款是否可被准确检索；观察主体—条款—法规之间的关系  
- Chat UI：发起条款提取、合同 A/B 对比、风险识别、法规引用等问题，由 Agent 调用 RAG 工具完成分析

**代表需求**：法律合同条款提取、风险审查、供应链合同对比、保险条款解读与合规性检查。

---

### 场景3：医疗与保险知识助手

**覆盖原始行业**：医疗/健康知识问答、保险理赔助手（面向终端用户和客服）  
**典型客户**：医院、健康管理公司、保险公司、医疗客服团队

**知识库内容示例**：
- 药品说明书、诊疗指南、医学科普文章
- 医疗保险条款、理赔案例、核保规则
- 医疗编码手册（如 ICD-10）、用药指南

**核心系统流程（Flow A）**：
- Admin UI：上传药品说明书、指南、保险条款等 → RAG Core 解析与索引  
- Chat UI：用户就症状、用药或保险保障范围提问 → Agent 使用 RAG 检索相关条款与说明 → Gemini 在回答中引用来源，并附带免责声明

**代表需求**：医疗问答机器人、保险保障范围查询、理赔政策解释等。

### 场景4：企业内部知识、培训与 HR 助手

**覆盖原始行业**：企业知识管理、培训教育智能辅导、HR 简历筛选、客户成功内部知识库  
**典型客户**：中大型企业、人力资源部门、培训机构、在线教育平台

**知识库内容示例**：
- 员工手册、制度流程、报销/审批规则
- 培训课件 PDF/PPT、课程字幕、练习题库、学习指南
- 岗位 JD、简历 PDF、面试评价表
- 内部 Wiki、最佳实践文档、培训材料

**核心系统流程（Flow A）**：
- Admin UI：按业务场景将制度、培训、简历等文档分批导入  
- Admin UI：用 Retrieval 测试“年假政策”“递归算法章节”“岗位匹配度”等关键词  
- Chat UI：
  - 员工/学员/HR 在对话中提问 → RAG 检索相关片段 → 输出解释、练习题或匹配评分

**代表需求**：入职培训助手、内部知识库、AI TA / HR 助手、培训问答机器人。

### 场景5：研究、财务与投资分析助手

**覆盖原始行业**：学术文献助手、财务报表分析、AI 会议纪要、投资研究与尽调  
**典型客户**：研究人员、高校、投研团队、财务部门、咨询公司、VC/PE 机构

**知识库内容示例**：
- 论文 PDF、研究笔记、参考书章节
- 年度/季度财报、审计报告、财务制度
- 会议记录转写、会议纪要、行动项清单
- 商业计划书、行业报告、尽调资料、历史投资案例

**核心系统流程（Flow A + Flow B）**：
- Admin UI：批量导入历史论文、财报、BP、会议记录 → 建立长期知识库  
- Chat UI：
  - 针对单次上传的会议记录做即时摘要（Flow B）  
  - 或在长期知识库上做跨文献问答、财务趋势分析、投资备忘录生成（Flow A）

**代表需求**：研究助理、财报分析机器人、投资尽调助手、会议纪要助手。

---

### 场景6：代码与技术文档助手

**覆盖原始行业**：技术文档代码助手、代码库智能助手  
**典型客户**：软件公司、开源项目、技术团队、DevRel 团队

**知识库内容示例**：
- 源代码（.py/.js/.ts/...）、README、API 规范
- 架构设计文档、设计图、变更日志、Issue/PR 记录

**核心系统流程（Flow A）**：
- Admin UI：导入一份代码仓快照 + 相关设计与 API 文档  
- Admin UI：用 Retrieval 检查某个类/函数名是否能被检索到  
- Chat UI：开发者提问“这个函数怎么用？修改这个模块会影响哪里？” → Agent 调用 RAG，返回解释、示例和依赖分析

**代表需求**：代码库问答、开发者 Onboarding 助手、文档生成与审查助手。

### 场景7：多语言、本地化与媒体内容问答助手

**覆盖原始行业**：多语言文档翻译问答、YouTube/播客内容问答、部分培训字幕场景  
**典型客户**：跨国企业、翻译公司、内容创作者、在线教育平台

**知识库内容示例**：
- 不同语言版本的产品/政策文档
- 多语言术语表（Excel）
- YouTube 字幕、播客转写、课程字幕、节目笔记

**核心系统流程（Flow A/B）**：
- Admin UI：导入多语言文档与术语表，或批量导入字幕文件  
- Chat UI：
  - 跨语言检索：用中文提问 → 检索英文文档 → 用中文回答  
  - 媒体问答：根据字幕回答“哪一期讲了某话题、对应时间戳是什么”等问题

**代表需求**：多语言知识库、本地化问答系统、视频/播客智能助手。

### 场景8：组合型企业知识与运营助手（高级版本）

**说明**：
- 将前面 7 类场景按企业真实需求进行组合，例如：
  - 一个大型制造企业同时需要：供应链助手 + 合同审查 + 内部知识库 + 多语言问答；
  - 一个政府/公共机构需要：政务问答 + 政策合规 + 培训/内部知识管理。
- 技术上仍然复用同一套 Admin UI + RAG Core + Agent Service + Chat UI，只是**按业务维度配置多个知识库与入口**。

**价值**：
- 用一套平台，覆盖企业在客户支持、内部知识、合规、投研等多个环节的 AI 需求，是高客单价、长期合作型项目的理想形态。

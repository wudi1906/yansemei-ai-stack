# AI 产品场景总览（基于 FastGPT + OneAPI + n8n）

> 本文档用于对外交付与销售，帮助你快速把现有技术栈打包成**可收费的标准产品方案**。

---

## 0. 当前基础设施概况

依托以下组件，你已经具备完整的 "AI 能力平台"：

- **FastGPT**（知识库与对话应用）  
  通过 `demo.yansemei.com` 对外提供知识库问答、聊天应用。
- **OneAPI**（模型中台）  
  通过 `api.yansemei.com` 聚合 DeepSeek 等模型，统一 Key 与配额管理。
- **n8n**（工作流 / RPA 自动化）  
  通过 `flow.yansemei.com` 构建各种 AI+自动化流水线。
- **Nginx Proxy Manager**（网关与 SSL）  
  管理所有域名与 HTTPS 访问。

下面的所有产品方案，都是基于这 4 个组件的不同组合。

## 1. 产品一：AI 周报 / 日报生成引擎

### 1.1 目标用户

- 职场人士、团队负责人、自由职业者。  
- 经常需要写工作日报 / 周报 / 项目复盘，但苦于耗时、难以总结。

### 1.2 解决的问题

- 把**零散的工作记录**（聊天记录、任务清单、便签）自动整理成：
  - 本周关键事项
  - 成果亮点
  - 问题与风险
  - 下周计划

### 1.3 技术实现

- **触发方式**：n8n `Chat Trigger` 或 `Webhook`。  
- **核心大脑**：`AI Agent` + `OpenAI Chat Model`（通过 OneAPI 调用 DeepSeek）。  
- **可选输出**：
  - 发送到邮件（Gmail / SMTP）
  - 发送到飞书 / 企业微信 / Slack
  - 追加到 Notion / Confluence 日志库

### 1.4 工作流结构

```text
[Chat Trigger / Webhook]
        │  （粘贴一周工作记录）
        ▼
[AI Agent]
  - System Prompt：资深团队负责人，擅长周报整理
  - User Prompt：用户粘贴的原始日志
        │
        ▼
[输出：Markdown 周报]
  ├─ 发送邮件 / IM
  └─ 存档到知识库
```

### 1.5 交付物

- 1 套在客户自有服务器或你的托管环境上的 n8n Workflow。  
- 1 份《如何整理输入日志》的用户操作指南（PDF/在线文档）。  
- 1 份示例周报模板（Markdown）。

## 2. 产品二：企业知识库 AI 客服（FastGPT Bot）

### 2.1 目标用户

- 有产品说明书、FAQ、培训资料的企业（SaaS、课程、线下服务）。
- 希望减少人工客服压力，提高 7x24 响应能力。

### 2.2 解决的问题

- 用户常见问题（价格、功能、退款、使用方法）由 AI 自动回答。  
- 新员工培训时，可直接向 Bot 询问公司制度、流程、产品细节。

### 2.3 技术实现

- **知识库构建**：
  - 使用 FastGPT 导入文档（PDF、Markdown、网页链接）。
  - 按产品/模块分类管理多个知识库。
- **模型层**：通过 OneAPI 调用 DeepSeek（或其它模型）。
- **接入方式**：
  - Web 小组件嵌入客户官网。  
  - 内部使用：仅内部员工可访问的管理后台或内网。

### 2.4 典型架构

```text
[用户浏览器]
    │  嵌入 FastGPT 前端
    ▼
[FastGPT 应用]
    │  根据对话调用：
    ▼
[OneAPI 模型层]
    │  访问 DeepSeek / 其他模型
    ▼
[返回答案 + 引用文档片段]
```

### 2.5 交付物

- 1 个配置完成的 FastGPT 应用（含知识库）。  
- 1 份嵌入代码 + FAQ 撰写模板。  
- 可选：与客户现有工单系统 / CRM 的对接方案（结合 n8n）。

## 3. 产品三：销售线索智能评分与分流

### 3.1 目标用户

- 有官网咨询表单 / Landing Page 的 B2B 或高客单价业务。  
- 销售团队人手有限，希望优先跟进“高价值线索”。

### 3.2 解决的问题

- 自动判断来访线索是否值得跟进，并给出 0–100 的评分。  
- 高分线索自动推送给销售负责人；低分线索自动归档。

### 3.3 技术实现

- **入口**：
  - Typeform / Google Form / 自建表单 → 通过 Webhook 推入 n8n。  
- **AI 评分**（n8n 工作流）：
  - 使用 `HTTP Request` 或 `AI Agent` 结合 OneAPI，给每条线索打分：
    - 预算区间
    - 职位/决策权
    - 需求紧急程度
- **分流动作**：
  - 高分：写入 CRM（HubSpot、Notion、Airtable…）+ 通知 Slack/飞书。  
  - 低分：写入“冷线索”表格，后续批量营销。

### 3.4 工作流结构

```text
[表单 Webhook]
    ▼
[AI 评分节点]
  - Prompt：请根据以下信息给出 0-100 分的成交概率，只输出数字
    ▼
[IF 条件判断]
  ├─ 分数 ≥ 80 → 写入 CRM + 发送 IM 通知
  └─ 分数 < 80  → 写入冷线索表
```

### 3.5 交付物

- 1 条完整的 n8n 工作流（含样例表单 / CRM 模板）。  
- 评分规则可配置的 Prompt 模板。  
- 使用手册：如何调整分数阈值、如何接入新的表单渠道。

## 4. 产品四：竞品情报雷达（Competitive Intelligence Radar）

### 4.1 目标用户

- 有直接竞争对手的 SaaS / 教培 / 内容型企业。  
- 需要持续关注竞品功能更新、价格调整、市场活动。

### 4.2 解决的问题

- 自动定时抓取竞品网站、博客、公众号、RSS。  
- 用 AI 总结“本周竞品做了什么”、“对我有何影响”。

### 4.3 技术实现

- **Schedule Trigger**：每天 / 每周定时触发。  
- **HTTP Request / RSS Read**：抓取竞品内容。  
- **AI 总结**：通过 OneAPI 调用 DeepSeek，总结变化点和潜在威胁。  
- **输出**：发送到 Slack/飞书群或邮件订阅。

### 4.4 工作流结构

```text
[Schedule Trigger]
    ▼
[HTTP Request 轮询竞品链接]
    ▼
[AI 总结（周报形式）]
    ▼
[Slack / 飞书 / Email 推送]
```

### 4.5 交付物

- 1 条可配置的情报工作流（可增删监控链接）。  
- 示例监控列表模板（Excel/Notion）。  
- 报告格式模板（Markdown 或 HTML 邮件）。

## 5. 产品五：SEO 内容自动生成工厂

### 5.1 目标用户

- 需要持续产出 SEO 文章的个人博主 / 内容团队 / 小型机构。  
- 想从“人肉写稿”升级为“AI 辅助 + 人工审核”。

### 5.2 解决的问题

- 根据关键词自动生成：文章标题、大纲、正文草稿。  
- 可选：自动发布到 WordPress / Ghost / 静态博客。

### 5.3 技术实现

- **输入来源**：
  - Google Sheets / Airtable 中存放关键词列表。  
- **工作流步骤**：
  1. 读取当天待写的关键词。  
  2. 调用 AI 生成大纲。  
  3. 基于大纲分段生成正文内容。  
  4. 可选：再用一个模型做语气统一、风格润色。  
  5. 通过 API 发布到博客平台。

### 5.4 工作流结构

```text
[Schedule Trigger]
    ▼
[读取关键词表]
    ▼
[AI 生成大纲]
    ▼
[AI 生成正文]
    ▼
[发布到博客 / 存档到 Notion]
```

### 5.5 交付物

- 1 套可运行的内容生产流水线（n8n 工作流 + 表格模板）。  
- 文章质量控制指南（如何人工审核与二次编辑）。
